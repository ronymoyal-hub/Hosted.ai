The Problem: Static GPU Allocation = 60% Waste

GPU workload utilization (industry standard):

  • Training: 30% idle (70% active) — long batch jobs with periodic peaks

Inference = lowest utilization (85% idle) = neocloud ROI declining as inference grows.

Result: Current GPU economics don't work. $3M in 80× H100 GPUs: not breaking even in 5 years.

The Solution: hosted·ai GPU Pool Virtualization

How hosted·ai Works: Spatial + Temporal Scheduling

Spatial Scheduling (Memory Isolation)

  • Virtualizes physical GPU cluster into GPU pools
  • Abstracts physical GPUs to logical resources (like vCPU in cloud)
  • Memory isolation ensures multi-tenant security (unlike NVIDIA MIG)

Temporal Scheduling (Time-Division)

  • Smart scheduler intercepts CUDA operations
  • Assigns GPU compute time to multiple workloads/customers in time slots
  • Kernel execution scheduled across time slots

Payload Caching in VRAM

  • Model weights persist in VRAM between slots (not reloaded from storage)
  • Fast context switching: ~100ms inference latency vs. ~300ms without caching

Contention Management

  • Soft resource caps per workload (prevent runaway processes)
  • QoS enforcement per customer SLA
  • Priority scheduling for premium tiers

Customer Experience: Fully Transparent

Customers see a normal GPU:

  • Full unmodified CUDA stack and standard NVIDIA tools
  • Kubernetes pods with systemd and remote SSH access
  • Full access to equivalent GPU resources without contention

Under the hood: hosted·ai scheduler manages time/space allocation across physical pool.
