One physical GPU, several paying tenants

hosted·ai virtualises a GPU fleet into pools and schedules tenant workloads across them, the way vCPU did for compute two decades ago. Customers see a normal GPU: unmodified CUDA, standard NVIDIA tooling, Kubernetes pods.

Spatial

Memory isolation

Physical GPUs become logical pools. Each tenant's VRAM is isolated, with optional hardware-assisted memory zeroing between sessions for regulated workloads.

Temporal

Time-division scheduling

Tenants get scheduled slots and burst to the full GPU inside them. Working sets stay cached in VRAM between slots, so context switches cost milliseconds, not model reloads.

Contention

QoS enforcement

Soft caps stop runaway processes, priority tiers map to your SLAs, and the scheduler smooths asynchronous peaks across tenants instead of letting one crowd out the rest.
